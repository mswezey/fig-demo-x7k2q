What's New
Now the team steers it — safely.
01

Plain forms, not code

Brian adds "ignore this vendor" the same way he'd fill out any web form. No engineer in the loop for routine changes.

02

A safety check before every save

The system replays the change against ~1,500 real FIG emails from recent weeks and shows exactly what would flip — "these 20 emails we used to flag would now be ignored" — before anyone can hit save.

03

Everything on the record

Every change logs who made it, what changed, and when. Six months later, "why did this stop flagging permit rejections?" always has an answer.
